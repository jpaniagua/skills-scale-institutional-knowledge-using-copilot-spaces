## Purpose
Ensure new team members have the context, access, and resources they need to contribute effectively to OctoAcme projects.

## How to Use
- Assign this checklist to each new team member
- Project Manager or team lead reviews and checks off items as completed
- Customize based on specific role and project needs
- Target completion: Within first 2 weeks

---

## General Onboarding (All Roles)

### Project Context
- [ ] Review Project One-pager and understand goals, metrics, stakeholders
- [ ] Read project charter or README in repo
- [ ] Understand success criteria and Definition of Done
- [ ] Review current sprint/iteration plan and backlog
- [ ] Meet with Project Manager for project overview
- [ ] Meet with Product Manager to understand product vision

### Tools & Access
- [ ] Access to project repository and codebase
- [ ] Access to project board (GitHub Projects, Jira, etc.)
- [ ] Access to communication channels (Slack, Teams, email lists)
- [ ] Access to documentation repository or wiki
- [ ] Calendar invites to relevant meetings (standups, planning, demos)
- [ ] Access to CI/CD pipelines and deployment tools
- [ ] Access to monitoring and observability dashboards

### Team & Process
- [ ] Introduced to all core team members
- [ ] Introduced to supporting role members
- [ ] Understand team communication norms and working hours
- [ ] Review escalation paths and who to contact for help
- [ ] Understand branching strategy and PR workflow
- [ ] Review code review guidelines and expectations
- [ ] Know where to find process documentation (this docs folder!)

---

## Role-Specific Onboarding

### For Developers
- [ ] Local development environment set up and tested
- [ ] Run project build and tests successfully
- [ ] Understand testing expectations (unit, integration, e2e)
- [ ] Review coding standards and style guides
- [ ] Access to necessary development tools and licenses
- [ ] Pair with experienced developer on first task
- [ ] Complete a "good first issue" to learn workflow

### For UX Designer
- [ ] Access to design tools (Figma, Sketch, etc.)
- [ ] Review existing design system and component library
- [ ] Understand user research findings and personas
- [ ] Review accessibility requirements and guidelines
- [ ] Meet with Product Manager to understand design priorities
- [ ] Access to user testing tools and feedback channels
- [ ] Review design review process and schedule

### For Technical Writer
- [ ] Access to documentation repositories and tools
- [ ] Review documentation style guide and templates
- [ ] Understand documentation release process
- [ ] Identify documentation priorities and gaps
- [ ] Access to product features for hands-on experience
- [ ] Meet with developers to understand technical depth needed
- [ ] Review how documentation is versioned and published

### For DevOps Engineer
- [ ] Access to cloud infrastructure (AWS, Azure, GCP)
- [ ] Access to CI/CD system and pipeline configurations
- [ ] Review deployment procedures and runbooks
- [ ] Access to monitoring and alerting systems
- [ ] Understand on-call rotation and incident response
- [ ] Review infrastructure-as-code repositories
- [ ] Access to secrets management and configuration systems

### For Business Analyst
- [ ] Understand business domain and key processes
- [ ] Review existing requirements documents and specs
- [ ] Identify key stakeholders and their needs
- [ ] Access to data analysis tools if applicable
- [ ] Review requirement templates and standards
- [ ] Understand approval and sign-off processes
- [ ] Meet with stakeholders to understand priorities

### For Scrum Master
- [ ] Understand current sprint cadence and ceremonies
- [ ] Review team velocity and historical metrics
- [ ] Access to project tracking tools
- [ ] Understand team's Agile maturity level
- [ ] Review retrospective notes from past sprints
- [ ] Identify current impediments and blockers
- [ ] Meet with PM and PdM to align on priorities

### For Security Lead
- [ ] Access to security scanning tools and reports
- [ ] Review threat model for the project/product
- [ ] Understand compliance requirements
- [ ] Access to security incident response procedures
- [ ] Review security policies and standards
- [ ] Identify upcoming security reviews or audits
- [ ] Meet with DevOps to understand security controls

---

## First Week Goals

### By End of Week 1:
- [ ] Attended all key team meetings
- [ ] Understand project goals and current status
- [ ] Can navigate codebase/documentation structure
- [ ] Know who to ask for help
- [ ] Completed at least one small contribution or review

---

## First Month Goals

### By End of Month 1:
- [ ] Contributing independently to role-specific work
- [ ] Participating actively in team ceremonies
- [ ] Building relationships with key collaborators
- [ ] Providing feedback on process and improvements
- [ ] Comfortable with team's workflow and tools
