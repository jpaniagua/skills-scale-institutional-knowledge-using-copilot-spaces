# Project Team Responsibility Checklist

## Purpose
This checklist helps ensure all critical responsibilities are assigned and accounted for across the project lifecycle. Use this during project initiation and planning to identify gaps and clarify ownership.

---

## Project Initiation Phase

### Project Definition & Strategy
- [ ] **Product Manager**: Define problem statement and success metrics
- [ ] **Business Analyst**: Document detailed business requirements
- [ ] **Product Manager**: Identify primary stakeholders
- [ ] **Project Manager**: Create project one-pager
- [ ] **Project Manager**: Develop initial timeline and milestones
- [ ] **Security Lead**: Assess security and compliance requirements
- [ ] **Technical Writer**: Scope documentation needs

### Resource Planning
- [ ] **Project Manager**: Identify required team roles and capacity
- [ ] **DevOps Engineer**: Assess infrastructure and tooling needs
- [ ] **UX Designer**: Determine design and research requirements
- [ ] **Project Manager**: Confirm team availability and commitment

---

## Project Planning Phase

### Requirements & Design
- [ ] **Business Analyst**: Facilitate stakeholder interviews
- [ ] **Product Manager**: Prioritize backlog and define features
- [ ] **Business Analyst**: Create detailed requirement specifications
- [ ] **UX Designer**: Conduct user research (if applicable)
- [ ] **UX Designer**: Create wireframes and prototypes
- [ ] **Security Lead**: Perform threat modeling
- [ ] **Developer**: Provide technical feasibility input

### Planning & Estimation
- [ ] **Scrum Master** (if Agile): Facilitate sprint planning
- [ ] **Project Manager**: Create project plan and schedule
- [ ] **Developer**: Estimate development effort
- [ ] **QA Engineer**: Draft test strategy and plan
- [ ] **DevOps Engineer**: Plan CI/CD pipeline setup
- [ ] **Technical Writer**: Create documentation plan

### Risk Management
- [ ] **Project Manager**: Create and maintain risk register
- [ ] **Security Lead**: Document security risks and mitigations
- [ ] **DevOps Engineer**: Identify infrastructure risks
- [ ] **All Team Members**: Contribute risk identification

---

## Execution & Tracking Phase

### Development & Implementation
- [ ] **Developer**: Implement features per acceptance criteria
- [ ] **Developer**: Write unit and integration tests
- [ ] **Developer**: Participate in code reviews
- [ ] **UX Designer**: Support design implementation
- [ ] **Technical Writer**: Create/update documentation alongside features

### Quality Assurance
- [ ] **QA Engineer**: Execute test plans and test cases
- [ ] **QA Engineer**: Report and track defects
- [ ] **QA Engineer**: Validate acceptance criteria
- [ ] **UX Designer**: Conduct usability testing
- [ ] **Security Lead**: Perform security testing and reviews

### DevOps & Infrastructure
- [ ] **DevOps Engineer**: Set up and maintain CI/CD pipelines
- [ ] **DevOps Engineer**: Configure monitoring and alerting
- [ ] **DevOps Engineer**: Manage deployment automation
- [ ] **Security Lead**: Configure security scanning in CI

### Project Coordination
- [ ] **Scrum Master** (if Agile): Facilitate daily standups
- [ ] **Project Manager**: Track progress and update status
- [ ] **Project Manager**: Update risk register weekly
- [ ] **Project Manager**: Coordinate cross-team dependencies
- [ ] **Scrum Master** (if Agile): Remove team impediments

---

## Risk & Communication Phase

### Communication Management
- [ ] **Project Manager**: Provide regular stakeholder updates
- [ ] **Product Manager**: Communicate product decisions
- [ ] **Technical Writer**: Prepare user-facing communications
- [ ] **Project Manager**: Maintain single source of truth for status

### Risk Monitoring
- [ ] **Project Manager**: Monitor and update risks
- [ ] **Security Lead**: Track security vulnerabilities
- [ ] **DevOps Engineer**: Monitor system health and performance
- [ ] **All Team Members**: Report new risks and blockers

### Escalation Handling
- [ ] **Scrum Master** (if Agile): Handle team-level blockers
- [ ] **Project Manager**: Escalate critical issues
- [ ] **Security Lead**: Lead security incident response
- [ ] **DevOps Engineer**: Lead infrastructure incident response

---

## Release & Deployment Phase

### Pre-Release Preparation
- [ ] **Developer**: Complete all feature PRs
- [ ] **QA Engineer**: Complete acceptance testing
- [ ] **Security Lead**: Verify security scans pass
- [ ] **Technical Writer**: Finalize release notes
- [ ] **DevOps Engineer**: Prepare deployment automation
- [ ] **DevOps Engineer**: Create rollback plan

### Deployment Execution
- [ ] **DevOps Engineer**: Execute staging deployment
- [ ] **QA Engineer**: Run smoke tests in staging
- [ ] **Security Lead**: Conduct security verification
- [ ] **DevOps Engineer**: Execute production deployment
- [ ] **DevOps Engineer**: Monitor post-deployment metrics

### Release Communication
- [ ] **Product Manager**: Announce release to stakeholders
- [ ] **Technical Writer**: Publish release documentation
- [ ] **Project Manager**: Update support team on changes

---

## Retrospective & Continuous Improvement Phase

### Retrospective Facilitation
- [ ] **Scrum Master** (if Agile) or **Project Manager**: Schedule retrospective
- [ ] **Scrum Master** (if Agile) or **Project Manager**: Facilitate retrospective meeting
- [ ] **All Team Members**: Participate and provide feedback
- [ ] **Scrum Master** (if Agile) or **Project Manager**: Document action items

### Knowledge Capture
- [ ] **Technical Writer**: Document project learnings
- [ ] **Project Manager**: Update process documentation
- [ ] **Developer**: Update technical documentation
- [ ] **Security Lead**: Document security lessons learned
- [ ] **DevOps Engineer**: Document infrastructure improvements

### Continuous Improvement
- [ ] **Project Manager**: Track action items to completion
- [ ] **All Team Members**: Implement improvement actions
- [ ] **Product Manager**: Measure outcome metrics
- [ ] **Team Leads**: Share learnings across organization

---

## Cross-Cutting Responsibilities

### Throughout Project Lifecycle
- [ ] **All Team Members**: Attend required meetings and ceremonies
- [ ] **All Team Members**: Maintain clear, timely communication
- [ ] **All Team Members**: Update status regularly
- [ ] **All Team Members**: Flag risks and blockers proactively
- [ ] **All Team Members**: Follow established processes and standards
- [ ] **All Team Members**: Support team members and collaborate effectively

### Leadership & Coordination
- [ ] **Project Manager**: Overall project coordination and delivery
- [ ] **Product Manager**: Product vision and prioritization
- [ ] **Scrum Master** (if Agile): Team facilitation and Agile coaching
- [ ] **Tech Lead** (if designated): Technical direction and architecture
