# OctoAcme — Role Mapping Worksheet

## Purpose
This worksheet helps project teams identify and assign roles for a project, ensuring accountability and coverage of all critical activities.

## How to Use
1. Review the project scope and deliverables
2. Check applicable roles from the lists below
3. Assign specific individuals or teams to each role
4. Identify any gaps in coverage
5. Revisit and update as the project evolves

## Role Assignment Decision Guide

### When to include UX Designer:
- ☐ Project has user-facing interfaces or features
- ☐ User experience is critical to success
- ☐ Accessibility requirements exist
- ☐ Design consistency needs to be maintained

### When to include Technical Writer:
- ☐ Public-facing documentation is needed
- ☐ API documentation is required
- ☐ Complex features need user guides
- ☐ Internal process documentation is part of deliverables

### When to include DevOps Engineer:
- ☐ Infrastructure changes are required
- ☐ New deployment pipelines needed
- ☐ Performance or scalability improvements planned
- ☐ Monitoring and observability enhancements needed

### When to include Business Analyst:
- ☐ Complex business requirements need decomposition
- ☐ Multiple stakeholder groups with varying needs
- ☐ Detailed workflow or process changes involved
- ☐ Data analysis or reporting requirements exist

### When to include Scrum Master:
- ☐ Team is using Agile/Scrum methodology
- ☐ Team needs process improvement support
- ☐ Multiple teams need coordination
- ☐ Team is new to Agile practices

### When to include Security Lead:
- ☐ Project handles sensitive or regulated data
- ☐ Authentication or authorization changes planned
- ☐ External integrations with security implications
- ☐ Compliance requirements (GDPR, SOC2, etc.)
