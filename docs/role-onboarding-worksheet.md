## Role-Specific Onboarding

### For Developers
- [ ] Access to code repositories and development environment
- [ ] Review branching strategy and PR conventions
- [ ] Set up local development environment
- [ ] Review Definition of Done and acceptance criteria standards
- [ ] Introduction to code review process
- [ ] Pair programming session scheduled

### For Product Managers
- [ ] Access to product roadmap and backlog tools
- [ ] Review product vision and current priorities
- [ ] Introduction to key stakeholders
- [ ] Review success metrics and KPIs
- [ ] Access to user research and analytics tools

### For Project Managers
- [ ] Access to project management tools (boards, tracking systems)
- [ ] Review current project portfolio and priorities
- [ ] Introduction to risk management process
- [ ] Review communication cadence and reporting templates
- [ ] Access to stakeholder contact list

### For QA Engineers
- [ ] Access to test environments and tools
- [ ] Review test strategy and automation framework
- [ ] Introduction to bug tracking and reporting process
- [ ] Review acceptance criteria standards
- [ ] Access to test data and environments

### For UX Designers
- [ ] Access to design tools and design system
- [ ] Review brand guidelines and design standards
- [ ] Introduction to user research process and tools
- [ ] Review design handoff process with developers
- [ ] Access to user feedback and analytics

### For Technical Writers
- [ ] Access to documentation repositories and tools
- [ ] Review documentation standards and style guide
- [ ] Introduction to documentation review process
- [ ] Access to subject matter experts
- [ ] Review content publishing workflow

### For DevOps Engineers
- [ ] Access to infrastructure and deployment systems
- [ ] Review CI/CD pipeline architecture
- [ ] Introduction to monitoring and alerting systems
- [ ] Review incident response procedures
- [ ] Access to infrastructure-as-code repositories

### For Business Analysts
- [ ] Access to requirements management tools
- [ ] Review requirement documentation standards
- [ ] Introduction to stakeholder interview process
- [ ] Access to business process documentation
- [ ] Review acceptance criteria definition process

### For Scrum Masters
- [ ] Review team's Agile practices and ceremonies
- [ ] Introduction to team members and dynamics
- [ ] Access to sprint planning and tracking tools
- [ ] Review impediment tracking process
- [ ] Access to retrospective and improvement documentation

### For Security Leads
- [ ] Access to security scanning and assessment tools
- [ ] Review security policies and compliance requirements
- [ ] Introduction to threat modeling process
- [ ] Review incident response procedures
- [ ] Access to security training materials

---

## General Onboarding Checklist
- [ ] Review OctoAcme Project Management Overview
- [ ] Review relevant process documentation for role
- [ ] Introduction to team members and key contacts
- [ ] Access to communication channels (Slack, Teams, etc.)
- [ ] Review meeting calendar and recurring meetings
- [ ] Set up 1:1s with manager/lead
- [ ] Review company policies and guidelines
- [ ] Access to relevant training and learning resources

---

## 30-60-90 Day Goals

### 30 Days
- Understand role responsibilities and expectations
- Complete initial onboarding checklist
- Shadow team members to learn processes
- Participate in team meetings and ceremonies

### 60 Days
- Begin taking ownership of assigned tasks
- Contribute to team deliverables
- Build relationships with cross-functional partners
- Provide initial feedback on processes

### 90 Days
- Full contribution to team goals
- Identify improvement opportunities
- Mentor newer team members (as applicable)
- Demonstrate proficiency in core responsibilities
